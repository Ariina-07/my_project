Ответы на вопросы по REST API

1. Какие основные принципы REST архитектуры вы знаете?

- Единообразие интерфейса - стандартные HTTP-методы
- Бессостояние - каждый запрос независим
- Кэшируемость- можно кэшировать ответы
- Клиент-сервер - разделение ответственности
- Слоистая система - можно добавлять промежуточные слои

2. В чем разница между HTTP методами PUT и PATCH?

- PUT - полное обновление ресурса (заменяет весь объект)
- PATCH - частичное обновление (изменяет только указанные поля)

3. Какие коды состояния HTTP наиболее часто используются в REST API?

- 200 - Успешный запрос
- 201 - Ресурс создан
- 400 - Неверный запрос
- 404 - Ресурс не найден
- 500 - Ошибка сервера

4. Что такое ресурсо-ориентированный дизайн URL?

URL представляют собой ресурсы (сущности), например:
- `/api/books` - коллекция книг
- `/api/books/1` - конкретная книга

5. Какие преимущества предоставляет пагинация в API?

- Уменьшает нагрузку на сервер
- Ускоряет ответ
- Удобство для клиента (постраничная навигация)

6. Как организовать валидацию данных в REST API?

- Проверка обязательных полей
- Проверка формата данных
- Проверка уникальности
- Возврат понятных ошибок

7. Какие подходы к обработке ошибок в REST API вы знаете?

- Единый формат ответов об ошибках
- Соответствующие HTTP-статусы
- Подробные сообщения об ошибках
- Логирование ошибок

8. В чем разница между query parameters и path parameters?

- Path parameters: `/api/books/1` (идентификатор ресурса)
- Query parameters: `/api/books?author=Толстой` (фильтрация, сортировка)
