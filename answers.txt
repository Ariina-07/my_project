Ответы на вопросы ПР1:

1. Три области работы с файлами в Git:
   - Рабочая папка - где мы редактируем файлы
   - Индекс (staging) - подготовленные файлы для коммита  
   - Репозиторий - история всех коммитов

2. Команды:
   - git add - добавляет файлы в индекс
   - git commit - сохраняет изменения в репозиторий
   - git push - отправляет изменения на удаленный репозиторий

3. Информативные сообщения помогают понять историю изменений.

4. git add имя_файла - добавляет конкретный файл, git add . - все файлы.

5. git status показывает текущее состояние файлов.

6. git remote add origin URL связывает с удаленным репозиторием.

7. Флаг -u устанавливает связь ветки с удаленным репозиторием.

8. Ветка по умолчанию - main.

9. git log показывает историю коммитов.

10. Без git add изменения не попадут в коммит.
