# Ответы на вопросы для самоконтроля

1.В чем преимущества использования миграций для управления схемой БД?
   - Позволяют версионировать изменения структуры БД
   - Легко откатывать изменения при необходимости
   - Автоматизируют процесс обновления схемы
   - Обеспечивают согласованность между разными средами (dev, prod)

2.Какие типы связей между таблицами вы знаете и как их организовать?
   - Один-ко-многим (one-to-many): через внешний ключ
   - Многие-ко-многим (many-to-many): через промежуточную таблицу
   - Один-к-одному (one-to-one): через внешний ключ с уникальным constraint

3.Что такое ORM и каковы его преимущества перед нативными SQL-запросами?
   - ORM (Object-Relational Mapping) - технология для работы с БД через объекты
   - Преимущества: безопасность от SQL-инъекций, переносимость между СУБД, более простой код

4.Как организовать пагинацию на уровне базы данных?
   - Использовать LIMIT и OFFSET в SQL-запросах
   - LIMIT определяет количество записей на странице
   - OFFSET - смещение для пропуска записей предыдущих страниц

5.Что такое транзакции и когда их следует использовать?
   - Транзакции - группа операций, выполняемых атомарно
   - Используются когда нужно обеспечить выполнение всех операций или ни одной
   - Пример: перевод денег между счетами

6.Какие методы оптимизации запросов к базе данных вы знаете?
   - Создание индексов для часто используемых полей
   - Оптимизация запросов (избегать SELECT *)
   - Кэширование результатов
   - Нормализация базы данных

7.Как организовать обработку ошибок при работе с БД?
   - Использовать try-catch блоки
   - Проверять результаты операций
   - Возвращать понятные сообщения об ошибках
   - Логировать ошибки для отладки

8.В чем разница между внутренним и внешним соединением таблиц?
   - INNER JOIN: возвращает только совпадающие записи из обеих таблиц
   - LEFT JOIN: возвращает все записи из левой таблицы и совпадающие из правой
   - RIGHT JOIN: возвращает все записи из правой таблицы и совпадающие из левой
