1. Основные проблемы legacy PHP-кода:
   - Смешение логики и представления
   - Уязвимости безопасности (SQL-инъекции, XSS)
   - Отсутствие структуры и ООП подхода

2. Преимущества Laravel:
   - Четкая структура MVC
   - Встроенные механизмы безопасности
   - ORM для работы с БД
   - Шаблонизатор Blade
   - Система миграций

3. MVC в Laravel:
   - Model - работа с данными (Eloquent)
   - View - шаблоны Blade
   - Controller - обработка запросов

4. Composer:
   - Менеджер зависимостей для PHP
   - Автозагрузка классов
   - Управление версиями пакетов

5. Возможности Blade:
   - Наследование шаблонов
   - Передача данных
   - Условные операторы и циклы
   - Компоненты

6. Маршрутизация в Laravel:
   - Определяется в файлах routes/
   - Поддержка различных HTTP методов
   - Параметры в маршрутах
   - Группировка маршрутов

7. Безопасность в Laravel:
   - Защита от SQL-инъекций (Eloquent)
   - CSRF-токены
   - Валидация данных
   - Экранирование вывода

8. Передача данных в шаблоны:
   - Через второй параметр view()
   - Использование with()
   - Через compact()
