1. HTML5-атрибуты валидации: required, type="email", type="tel", pattern, minlength
2. novalidate отключает встроенную валидацию, JavaScript дает больше контроля
3. label связывается с input через for и id, важно для доступности и usability
4. email - для почты, tel - телефоны, password - пароли, date - даты
5. pattern использует регулярные выражения для сложной валидации
6. События: input, blur, change. input - в реальном времени, blur - при уходе с поля
7. Сообщения об ошибках должны быть видимыми и доступными для скринридеров
8. fieldset и legend группируют связанные поля, улучшая структуру
9. Кастомная валидация через JavaScript для сложных проверок (пароли, логика)
10. CSS псевдоклассы :valid, :invalid и кастомные классы .valid, .invalid
